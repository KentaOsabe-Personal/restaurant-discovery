# Brief: ramen-omakase

## 課題

ラーメン検索では自分で条件を考えて入力する必要があり、「今日どこのラーメン行こう？」という気軽な使い方ができない。
既存の居酒屋おまかせのように、ボタン一つでAIにラーメン屋を推薦してほしい。

## 現状

- 居酒屋おまかせ（OmakaseService）が4エリアで稼働中
- ラーメンタブは自然文検索 + 距離フィルターが動作（前提: Phase 1, 2 実装済み）
- ラーメンタブにはおまかせボタンが未実装

## 望む結果

- ラーメンタブにおまかせボタンがあり、距離選択と組み合わせて使える
- 「車で30分以内のラーメン」のようなおまかせ体験ができる
- 新潟周辺のラーメン激戦区からランダムにエリアが選ばれ、AI推薦される

## アプローチ

既存の OmakaseService パターンを踏襲し、ラーメン用のエリアマスターデータ（新潟市内のラーメン激戦区、燕三条方面等）を定義する。距離フィルター（Phase 2）の圏内からエリアをランダム選定し、Google Places → AI推薦のパイプラインに流す。

## スコープ

- **含む**:
  - ラーメンタブにおまかせボタンを追加（距離選択との組み合わせ）
  - ラーメン用エリアマスターデータの定義（東区・西区・燕三条・長岡生姜醤油エリア等）
  - RamenOmakaseService（or OmakaseService拡張）: 距離圏内のエリアからランダム選定
  - おまかせパイプライン: エリア選定 → Google Places → AI推薦
  - `/api/omakase` に `mode: "ramen"` + 距離パラメータ対応
- **含まない**:
  - ラーメンのサブジャンル別おまかせ（味噌・醤油・塩等の指定）
  - エリアマスターのUI管理（ハードコードで十分）
  - 居酒屋おまかせの変更

## バウンダリ候補

- フロントエンド: おまかせボタンUI + 距離選択との連携
- バックエンド: ラーメン用エリアマスター + おまかせロジック
- API契約: `/api/omakase` の mode + distance 対応

## スコープ外（明示的非目標）

- 居酒屋おまかせ機能への変更
- ラーメン店のジャンル別絞り込み（つけ麺、まぜそば等）
- エリアマスターデータの動的管理

## 上流 / 下流

- **上流**: ramen-search-mode（タブUI・mode基盤）、distance-filter（距離フィルターとの組み合わせ）
- **下流**: なし（現時点での最終フェーズ）

## 既存Specとの接点

- **拡張対象**: omakase-button / omakase-4areas（パターン踏襲、ラーメン版を追加）
- **隣接**: distance-filter（距離圏内からのエリア選定）

## 制約

- 既存の居酒屋おまかせ（OmakaseService）のパターンを踏襲
- ラーメン特有のサブエリア（燕三条、長岡生姜醤油エリア等）を定義
- 距離フィルターとの組み合わせが前提
