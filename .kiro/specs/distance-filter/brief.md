# Brief: distance-filter

## 課題

ラーメン検索で新潟市外の店舗も大量にヒットし、実際に車で行ける範囲の店に絞れない。
居酒屋は繁華街限定で問題ないが、ラーメンは車移動前提のため距離ベースのフィルターが必要。

## 現状

- ramen-search-mode で基本的なラーメン検索が動作（前提: 先に実装済み）
- Google Places API は `locationBias`（circle: center + radius）をサポート
- 自宅の緯度経度を設定する仕組みがない
- 検索結果に距離情報が含まれていない

## 望む結果

- ラーメンタブに「車でN分以内」の距離選択UIがある
- 検索結果が指定距離内の店に絞られる
- PlaceCardに自宅からの概算距離が表示される
- 自宅の緯度経度がconfigファイルで管理されている

## アプローチ

フロントエンドに自宅の緯度経度をconfigファイルとして定義し、距離選択ボタン（15分/30分/1時間）をラーメンタブに追加する。バックエンドは受け取った距離パラメータをGoogle Places APIの `locationBias` (circle) に変換し、起点+半径で検索する。レスポンスに自宅からの直線距離（Haversine）を含める。

## スコープ

- **含む**:
  - `frontend/src/config/homeLocation.ts` に自宅の緯度経度を定義
  - ラーメンタブに距離選択UI（15分 / 30分 / 1時間 のボタン or セレクタ）
  - 距離をAPIリクエストパラメータとして送信
  - PlaceCardに自宅からの概算距離を表示
  - バックエンド: Google Places API の `locationBias` (circle) を活用した起点+半径検索
  - 車N分 → 半径kmの変換ロジック（15分≒15km、30分≒30km、1時間≒60km 目安）
  - レスポンスに自宅からの直線距離を含める
- **含まない**:
  - Google Maps Directions API による正確な経路計算
  - 地図上の検索圏円表示（オプション、スコープ外）
  - 居酒屋タブへの距離フィルター適用

## バウンダリ候補

- フロントエンド: config定義 + 距離選択UI + 距離表示
- バックエンド: locationBias変換 + Haversine距離計算
- API契約: distance（分単位）パラメータの追加

## スコープ外（明示的非目標）

- 正確な車の移動時間の算出
- 複数の起点（自宅以外）の対応
- 居酒屋モードへの距離フィルター適用

## 上流 / 下流

- **上流**: ramen-search-mode（タブUI・mode基盤）
- **下流**: ramen-omakase（距離フィルターとおまかせの組み合わせ）

## 既存Specとの接点

- **拡張対象**: ramen-search-mode（距離パラメータをAPIリクエストに追加）
- **隣接**: google-places-service（locationBias パラメータ追加）

## 制約

- 直線距離の概算で十分（個人利用のため）
- 車の移動時間は直線距離ベースの目安変換（Haversine公式）
- 自宅位置はハードコードで管理（UIからの変更機能は不要）
