# Requirements Document

## Project Description (Input)
現在のアプリは居酒屋・バー検索に特化しており、ラーメン屋を探したいときに使えない。ユーザー（個人利用）は新潟県内のラーメン屋を自然文で検索し、AI推薦を受けたい。画面上部のタブで「居酒屋・バー」と「ラーメン」を切り替えられるようにし、ラーメンタブで自然文検索するとジャンル「ラーメン」が自動付与されて検索される。AIがラーメンの特徴（味の系統・麺の太さ等）を考慮して推薦理由を提示する。既存機能（フィードバック再レコメンド・食べログリンク・検索履歴・地図表示）がラーメンタブでもそのまま動作する。

## Boundary Context

- **In scope**: タブUIによるモード切り替え、ラーメン向け自然文検索・AI推薦、既存機能（フィードバック再レコメンド・食べログリンク・地図表示・もっと見る・検索条件タグ）のラーメンタブ対応、検索履歴のタブ別管理
- **Out of scope**: 距離フィルター（Phase 2 distance-filter で対応）、ラーメンおまかせ機能（Phase 3 ramen-omakase で対応）、自宅位置の設定（Phase 2 で対応）、ラーメンのサブジャンル別フィルター
- **Adjacent expectations**:
  - 既存の検索パイプライン（QueryParserService → GooglePlacesService → RecommendationService）がラーメンモード用にプロンプト・パラメータを切り替え可能であること
  - 既存の再レコメンド機能がモードを引き継ぎ、ラーメンモードでも再レコメンドが動作すること
  - 既存の居酒屋・バー検索およびおまかせ機能は影響を受けないこと

## Requirements

### Requirement 1: タブ切り替えUI

**Objective:** ユーザーとして、画面上部のタブで「居酒屋・バー」と「ラーメン」を切り替えたい。検索対象を明確に区別し、それぞれに適した検索体験を得るため。

#### Acceptance Criteria
1. When アプリを起動した時, the system shall 「居酒屋・バー」タブをデフォルトで選択状態にする
2. When ユーザーが「ラーメン」タブをクリックした時, the system shall アクティブタブを「ラーメン」に切り替え、検索結果・入力中のクエリ・検索条件タグ・フィードバック状態をリセットする
3. When ユーザーが「居酒屋・バー」タブをクリックした時, the system shall アクティブタブを「居酒屋・バー」に切り替え、検索結果・入力中のクエリ・検索条件タグ・フィードバック状態をリセットする
4. While 「居酒屋・バー」タブがアクティブな時, the system shall おまかせボタン（4エリア）を表示する
5. While 「ラーメン」タブがアクティブな時, the system shall おまかせボタンを非表示にする

### Requirement 2: ラーメン自然文検索

**Objective:** ユーザーとして、ラーメンタブで自然文のクエリを入力してラーメン屋を検索したい。「こってり味噌ラーメン」「駅前で安いラーメン」のような自然な表現で店を探せるようにするため。

#### Acceptance Criteria
1. While 「ラーメン」タブがアクティブな時, When ユーザーが検索クエリを送信した時, the system shall ジャンル「ラーメン」を検索条件に自動付与して検索を実行する
2. While 「ラーメン」タブがアクティブな時, When 検索クエリにユーザーがジャンルを明示的に含めなかった場合でも, the system shall ジャンルに「ラーメン」を設定して検索する
3. While 「ラーメン」タブがアクティブな時, When 検索クエリにエリア・価格帯・キーワードが含まれている場合, the system shall それらの条件をジャンル「ラーメン」と組み合わせて検索する
4. When ラーメンモードで検索が完了した時, the system shall 検索条件タグにジャンル「ラーメン」を含めて表示する

### Requirement 3: ラーメンAI推薦

**Objective:** ユーザーとして、ラーメン屋の推薦時にラーメン固有の特徴（味の系統・麺の太さ・スープの種類など）を考慮した推薦理由を受け取りたい。より的確な店選びができるようにするため。

#### Acceptance Criteria
1. When ラーメンモードで検索結果が返された時, the system shall 推薦理由にラーメン固有の特徴（味の系統、麺の太さ、スープの種類など）を含めて提示する
2. When ラーメンモードで推薦を生成する時, the system shall 既存の推薦と同じ形式（推薦店リスト＋推薦理由＋その他候補）で結果を返す

### Requirement 4: 既存機能のラーメンタブ対応

**Objective:** ユーザーとして、ラーメンタブでも居酒屋・バータブと同等の補助機能（フィードバック再レコメンド・食べログリンク・地図表示・もっと見る）を利用したい。モードに関係なく一貫した操作体験を得るため。

#### Acceptance Criteria
1. While 「ラーメン」タブがアクティブで検索結果が表示されている時, When ユーザーがフィードバックを送信した時, the system shall ラーメンモードのコンテキストを維持したまま再レコメンドを実行する
2. While 「ラーメン」タブがアクティブな時, When 検索結果が表示された時, the system shall 各店舗に食べログ検索リンクを表示する
3. While 「ラーメン」タブがアクティブな時, When 検索結果が表示された時, the system shall 地図上に検索結果のマーカーを表示する
4. While 「ラーメン」タブがアクティブな時, When 検索結果が表示された時, the system shall AI推薦に含まれなかったその他候補を「もっと見る」セクションで表示する
5. While 「ラーメン」タブがアクティブな時, When 検索結果が表示された時, the system shall 検索条件タグ（エリア・ジャンル・価格帯・キーワード）を表示する

### Requirement 5: 検索履歴のタブ別管理

**Objective:** ユーザーとして、タブごとに検索履歴を分けて管理したい。居酒屋の検索履歴とラーメンの検索履歴が混在しないようにするため。

#### Acceptance Criteria
1. While 「ラーメン」タブがアクティブな時, the system shall ラーメンタブの検索履歴のみを履歴チップとして表示する
2. While 「居酒屋・バー」タブがアクティブな時, the system shall 居酒屋・バータブの検索履歴のみを履歴チップとして表示する
3. When ラーメンタブで検索を実行した時, the system shall そのクエリをラーメンタブの検索履歴に追加する
4. When 居酒屋・バータブで検索を実行した時, the system shall そのクエリを居酒屋・バータブの検索履歴に追加する
5. The system shall 既存の検索履歴（ラーメンモード導入前に保存されたもの）を居酒屋・バータブの履歴として引き続き表示する

### Requirement 6: 後方互換性

**Objective:** 既存ユーザーとして、居酒屋・バーの検索機能が従来どおり動作し続けてほしい。ラーメンモードの追加によって既存の体験が壊れないようにするため。

#### Acceptance Criteria
1. While 「居酒屋・バー」タブがアクティブな時, the system shall 従来と同一の検索・推薦・おまかせ・フィードバック再レコメンド動作を提供する
2. When モードを指定せずに検索した時, the system shall 従来の居酒屋・バー向け動作をデフォルトとして実行する
3. The system shall 居酒屋・バータブのすべての既存機能（自然文検索・AI推薦・おまかせ・フィードバック再レコメンド・食べログリンク・地図表示・検索履歴・検索条件タグ・もっと見る）を変更なく維持する
