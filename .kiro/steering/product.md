# プロダクト概要

レストラン発見・探索プラットフォーム。ユーザーが新しい飲食店を見つけ、情報を整理・共有できるWebアプリケーション。

## コア機能

- **自然文レストラン検索**（実装済み）: 「渋谷でリーズナブルなイタリアン」のような自然文クエリで店舗を探せる
- **AIレコメンデーション**（実装済み）: OpenAIが候補店から最適な3〜5件を選定し、推薦理由を提示
- **フィードバック再レコメンド**（実装済み）: 初回推薦に対して追加要望を自然文で伝えると、条件をマージしておすすめを再選定できる
- **おまかせ機能**（実装済み）: 新潟4エリア（駅前・駅南・古町・長岡）からランダムにサブエリアを選び、夜の居酒屋・バーを推薦
- **地図表示**（実装済み）: 検索結果をGoogle Maps上にマーカー表示（`@vis.gl/react-google-maps`）
- **検索履歴**（実装済み）: 過去の検索クエリをlocalStorageに保存・チップ表示
- **検索条件タグ**（実装済み）: 現在の検索条件（エリア・ジャンル・価格帯）をタグ表示
- **もっと見る**（実装済み）: AIが選ばなかった他の候補店を展開表示
- **食べログリンク**（実装済み）: 店舗名から食べログ検索URLを自動生成
- **検索モード切り替え**（実装済み）: 「居酒屋・バー」と「ラーメン」をタブで切り替え、同じ検索体験の中でモード別の条件を扱う
- **ラーメン検索**（実装済み）: ラーメンモードではジャンルを「ラーメン」に固定し、味・種類はキーワードとして抽出して推薦精度を高める
- **距離フィルター**（実装済み）: ラーメン検索では自宅起点の概算距離を使い、車移動時間帯（30分以内・1時間以内・1〜2時間）で候補を絞る
- **ラーメンおまかせ**（実装済み）: 距離条件に合う新潟県内のラーメン激戦区からランダムにエリアを選び、推薦と候補を返す
- ユーザーによるレビュー・評価（予定）

## AI検索パイプライン

```
自然文クエリ
  → QueryParserService（OpenAI: エリア/ジャンル/価格帯を抽出）
  → GooglePlacesService（Google Places API: 候補店を取得）
  → RecommendationService（OpenAI: 最適な店を推薦・理由付け）
  → フロントエンド表示（地図・カード・検索条件タグ）
```

### おまかせパイプライン（夜専用）

```
エリアID（ekimae / ekinan / furumachi / nagaoka）
  → OmakaseService（サブエリアをランダム選択）
  → GooglePlacesService（候補店を取得）
  → アドレスフィルタリング（サブエリア名でふるい掛け）
  → RecommendationService（5件固定で推薦）
  → フロントエンド表示
```

### 再レコメンドパイプライン

```
初回検索結果 + ユーザーフィードバック
  → QueryParserService（追加要望から差分条件を抽出）
  → 既存条件とマージ
  → GooglePlacesService（再取得）
  → RecommendationService（元クエリ文脈を保って再推薦）
  → フロントエンド表示
```

### ラーメン検索パイプライン

```
自然文クエリ + mode=ramen + travel_time（任意）
  → QueryParserService（OpenAI: エリア/価格帯/味・種類キーワードを抽出、ジャンルはラーメン固定）
  → GooglePlacesService（Google Places API: ラーメン向けテキストクエリで候補店を取得）
  → DistanceCalculatorService（自宅起点の概算距離を付与）
  → travel_time がある場合は距離帯でフィルタリング
  → RecommendationService（ラーメン向け選定基準で推薦・理由付け）
  → フロントエンド表示（距離・地図・カード・検索条件タグ）
```

### ラーメンおまかせパイプライン

```
mode=ramen + travel_time（任意）
  → RamenOmakaseService（距離条件に合う激戦区をランダム選択）
  → GooglePlacesService（選択エリアのラーメン候補を取得）
  → アドレスフィルタリング（選択エリア名でふるい掛け）
  → DistanceCalculatorService（自宅起点の概算距離を付与）
  → RecommendationService（ラーメン向け選定基準で推薦）
  → フロントエンド表示
```

## 主要ユースケース

- 自然文で条件（エリア・ジャンル・価格帯）を指定してレストランを探す
- AIが選んだおすすめ店と推薦理由を確認して店を選ぶ
- 初回結果を見たあと、「もう少し静かなお店」などの追加要望で候補を絞り直す
- 4エリアボタンから夜の飲み屋をおまかせで発見する
- タブを切り替えてラーメン店を探し、車移動時間の目安で候補を絞る
- 距離条件だけ指定してラーメン激戦区からおまかせ推薦を受ける
- 行きたい店・行った店を記録・管理する（予定）

## 価値提案

シンプルで直感的なUIで、レストラン探しの体験を向上させる。フロントエンドとバックエンドを疎結合に保ち、将来的な機能拡張に対応しやすい設計。
